os-organization

Posted on 2021-12-30 In 操作系统 Disqus:

概述

操作系统满足3个要求：多路复用，隔离，交互

侧重于以宏内核为中心的主流设计

Abstracting physical resources

程序直接操作资源

如果应用程序直接操作资源会怎样？应用程序根据自己的需求定制库，直接与硬件资源交互，并以应用程序的最佳方式使用这些资源。一些嵌入式设备或实时系统的操作系统就是这样组织的。

性能更好，但风险更高，实现更难。程序不能有bug，一旦出 bug 会影响所有所有程序。必须定期放弃 cpu，以便其他程序运行。内存操作，可能会覆盖其他程序的数据。

资源抽象

为了实现强隔离， 禁止应用程序直接访问敏感的硬件资源，而是将资源抽象为服务。

文件系统，通过 open、read、write和close系统调用与存储交互，而不是直接读写磁盘。提升操作便利性和安全性

虚拟内存，隔离各个进程的内存。

cpu 调度，实现进程共享 cpu

资源隔离

user/kernel mode：cpu 实现隔离

Virtual Memory：mmu 实现隔离

User/supervisor mode and system calls

CPU为强隔离提供硬件支持。例如，RISC-V有三种CPU可以执行指令的模式：机器模式(Machine Mode)、用户模式(User Mode)和管理模式(Supervisor Mode)。在机器模式下执行的指令具有完全特权；CPU在机器模式下启动。机器模式主要用于配置计算机。Xv6在机器模式下执行很少的几行代码，然后更改为管理模式。

应用程序只能执行用户模式的指令（如 add mov），并被称为在用户空间中运行，而此时处于管理模式下的软件可以执行特权指令（如 中断 页表 ），并被称为在内核空间中运行。在内核空间（或管理模式）中运行的软件被称为内核。

怎么判断处理 user mode 还是 kernel mode

cpu 有个标志位，1 表示 user mode

如何进入 kernel mode

xv6中调用 ecall(sysCallNo) 就可以跳转到 kernel 执行，入参 sysCallNo 为对应系统调佣的编号。kernel 根据 sysCallNo 找到对应的 system call 并执行。

微内核与宏内核

宏内核（monolithic kernel）：整个操作系统都驻留在内核中，所有系统调用的实现都以管理模式运行

微内核（microkernel）：操作系统最基本的部分驻留在内核中，减少内核代码量，降低内核出错风险

如上图，文件系统作为进程服务 File server 在用户空间，shell 想要访问文件需要与 File server 进程通信。进程间通信需要经过内核。相比于宏内核，多了一次 mode 切换，因此微内核性能较低。一般操作系统都是宏内核

进程概述

进程 vs 线程

进程是资源调度的基本单位，线程是 CPU 调度的基本单位。

进程抽象给程序提供了一种错觉，即它有自己的专用机器。linux 使用页表（page table，由硬件实现）为每个进程提供自己的地址空间，页表将虚拟地址转换为物理地址

每个进程都有一个执行线程（简称线程）来执行进程的指令。一个线程可以挂起并且稍后再恢复。内核通过挂起当前线程，并恢复另一线程来实现 cpu 资源共享。

线程的大部分状态（本地变量、函数调用返回地址）存储在线程的栈区上。每个进程有两个栈区：一个用户栈区和一个内核栈区（p->kstack）。当进程执行用户指令时，只有它的用户栈在使用，它的内核栈是空的。当进程进入内核（由于系统调用或中断）时，内核代码在进程的内核堆栈上执行；当一个进程在内核中时，它的用户堆栈仍然包含保存的数据，只是不处于活动状态。进程的线程在主动使用它的用户栈和内核栈之间交替。内核栈是独立的（并且不受用户代码的保护），因此即使一个进程破坏了它的用户栈，内核依然可以正常运行。

进程属性由 proc 结构体表示，p->xxx来引用proc结构体的元素。 p->state表明进程是已分配、就绪态、运行态、等待I/O中（阻塞态）还是退出。p->pagetable，指向该进程页表的指针

启动 XV6 和第一个进程

boot loader

机器通电后，rom 中 boot loader 将 xv6 kernel 加载到内存中

machine mode

cpu 在机器模式下从_entry (kernel/entry.S:6)开始运行xv6。_entry的指令设置了一个栈区，这样 xv6 就可以运行 C 代码。

_entry便调用 C 代码start(kernel/start.c:21)。函数start执行一些仅在机器模式下允许的配置，如对时钟芯片进行编程以产生计时器中断。

完成后切换到管理模式，RISC-V 提供指令mret以进入管理模式

supervisor mode

进入管理模式。这将导致程序计数器（PC）的值更改为main(kernel/main.c:11)函数地址。在main 初始化几个设备和子系统后，便通过调用userinit (kernel/proc.c:212)创建第一个进程，第一个进程执行一个用 RISC-V 程序集写的小型程序：initcode. S (**user/initcode.S:**1)，它通过调用exec系统调用重新进入内核。

exec用一个新程序（如 /init）替换当前进程的内存和寄存器。一旦内核完成exec，它就返回/init进程中的用户空间。如果需要，init(user/init.c:15)将创建一个新的控制台设备文件，然后以文件描述符0、1和2打开它。然后它在控制台上启动一个shell。系统就这样启动了。
