Quarkus框架介绍

Posted on 2022-11-10 In 框架 Views: Disqus:
Quarkus 是一种用于构建云原生应用的 Java 框架，它被设计成轻量级、快速启动和内存占用小，适合构建服务器端和云原生应用程序。使用GraalVM技术，将java代码转换为本机二进制，从而实现更高的执行速度和更小的应用程序体积。支持开箱即用的k8s部署，支持 Kotlin 和 Scala。

什么是 Quarkus

  • 是一个 Java 应用框架，对 Springer 友好。

    • 依赖注入基于 CDI，使用 JAX-RS 注解定义 REST 端点，JPA 实现 ORM、JTA 声明事务

    • Spring代码

    • Quarkus 代码

  • 容器优先、微服务优先

    • MicroProfile 规范监控，⽀持 Prometheus、Grafana
    • 基于Graalvm构建native镜像，从而实现更高的执行速度和更小的应用程序体积
    • 使⽤Kubernetes API，访问 ConfigMap 或 Secret

  • 可与常⽤Java标准、框架和库系统⼯作

    • Redis，Kafka，JPA，Vert.x

  • 同时⽀持命令式和响应式代码

云原生时代 Java 面临的挑战

  • 沙箱优势不再，Docker 已实现⼀次编译到处运⾏
  • JDK ⽐应⽤还⼤
  • 构建的镜像中超过⼀半的⽆⽤代码和依赖
  • 启动速度慢
    • 启动 Docker 容器 > 启动JVM > 加载 Java 应⽤
  • 内存占⽤⼤
  • 空 Java Web 应⽤ 100M，空 Go Web 应⽤ 10M 左右

镜像

  • Alpine 作为最基础 Linux 镜像，仅 5M 左右
  • 使⽤ JDK9 开始⽀持的模块化特性构建基础镜像
  • jlink简化jre

结论：经过上述策略，可以将 Java 镜像瘦身一半以上

启动

  • JIT编译器，初始化+预热耗时⻓
  • 单体服务架构，引⼊依赖更多，预热耗时更⻓

内存

通过如下命令可以解析出 java 进程的内存占用：

java -XX:NativeMemoryTracking=summary -Xmx512m -Xmn512m -jar spring-demo-0.0.1-SNAPSHOT.jar jcmd pid VM.native_memory summary scale=MB

GraalVM AOT

https://www.infoq.com/articles/native-java-graalvm/

  • Points-to Analysis
    • classes, methods, fields, and libraries
  • Run Initializations at build time
    • classes are safe to initialize
  • Heap snapshotting
    • Java objects allocated by static initializers, and all the objects that are reachable
    • application starts much faster with a pre-populated heap
  • Substrate VM
    • memory management and thread scheduling
    • compile into native code

Quarkus&GraalVM

  • 构建时处理
    • 构建时做传统框架在运⾏时做的事情：配置解析、classpath 扫描、基于classloading的功能切换
    • ⽬标：应⽤程序只包含在运⾏时使⽤的类
  • 减少反射的使⽤
    • Java的动态性使其具有了很⾼的⽣产⼒，同时也使编译器⽆ 法掌握应⽤程序的⾏为，并限制了优化程度
    • Quarkus构建时解析了每个依赖关系，以常规调⽤取代反射 调⽤。使⽤⾃定义代理，代替动态代理

命令式 & 响应式

  • 基于 Vert.x ，同时⽀持命令式和响应式
  • 在命令式下，I/O线程和业务线程切换
  • 在响应式下，都在I/O线程下处理。适⽤于I/O密集型服务， 例如微服务⽹关
