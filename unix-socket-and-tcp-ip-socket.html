Unix socket 与 TCP/IP socket 的区别

Posted on 2021-11-25 In 计算机网络 Views: Disqus:
Unix socket 又叫 IPC（inter-process communication 进程间通信） socket，基于文件系统实现同一主机上的进程间通信。TCP/IP socket 基于网络系统实现进程间网络通信。socket 是 Berkeley Software Distribution (BSD) 定义的通信标准接口。

Unix socket

又叫 IPC（inter-process communication 进程间通信） socket，用于实现同一主机上的进程间通信。通过监听 path（Unix 文件系统路径） 而不是 port。直接将应用层数据从一个进程拷贝到另一个进程，不需要经过网络层（检查、路由等），因此比 IP socket 效率更高。

MySQL 对此特性的优化，如果发现地址是 localhost，就会使用 unix 套接字来提升性能。具体参考：https://dev.mysql.com/doc/refman/8.0/en/connecting.html

通过以下命令来列出本机 Unix socket

Linux: netstat -a -p --unix

Mac: netstat -a -f unix

Windows: 没有 Unix socket，有个类似的机制叫做 Named pipes。win10 支持 Unix socket，有一些限制，具体参考：https://devblogs.microsoft.com/commandline/af_unix-comes-to-windows/

TCP/IP socket

进程间网络通信机制，也可以实现同一主机进程通信（通过使用环回接口）。一个 IP socket connection 由 <local IP, local port, remote IP and remote port> 唯一标识。
