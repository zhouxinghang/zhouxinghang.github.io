操作系统页表

Posted on 2022-01-05 In 操作系统 Views: Disqus:

概述

页表是操作系统为每个进程提供自己私有地址和内存空间的机制。页表维护了虚拟地址与物理地址的映射，每个进程维护一个页表，描述每个进程的用户地址空间。

页表硬件

指令（用户和内核指令都包括）使用的是虚拟地址，而机器的物理内存是由物理地址索引的。页表硬件通过将每个虚拟地址映射到物理地址来为这两种地址建立联系。

mmu 硬件概述

XV6基于Sv39 RISC-V运行，只使用低 39bit，高 25bit 不使用。中间 27bit 索引页表条目（Page Table Entries/PTE），最多 2^27 个 PTE。每个 PTE 包含一个 44bit 的物理页码（Physical Page Number/PPN）和一些标志 Flags。低 12bit 是物理地址偏移量。PPN+Offset 组成物理地址。简化地址映射如下

page table 地址什么维护？

satp 寄存器保存进程的起始地址，指向该进程 Page table 的物理地址。进程切换时，satp 会变

完整的地址映射

实际的转换分三个步骤进行。页表以三级的树型结构存储在物理内存中。该树的根是一个4096字节的页表页，其中包含512个PTE，其中包含该树下一级页表页的物理地址。这些页中的每一个都包含该树最后一级的512个PTE（也就是说每个PTE占8个字节，只使用前54bit，正如图3.2最下面所描绘的）。分页硬件使用27位中的前9位在根页表页面中选择PTE，中间9位在树的下一级页表页面中选择PTE，最后9位选择最终的PTE。

红色箭头：satp 指向 L2 页 PA（physical address），L2 页 PPN 指向 L1 页 PA，L1 页 PPN 指向 L0 页 PA。L0 页 PPN + Offset = 物理内存 PA 。

蓝色箭头：VA 中的 L2 索引 L2 页的 PTE，L1 索引 L1 页的 PTE。L0 索引 L0 页的 PT

PTE：页表项，64bit，低 10bit 用于权限控制等标志位，中间 44bit PPN 指向下一页 PA

三级页作用：减少内存占用，索引效率会变低。TLB（translation lock-aside buffer）缓存 VA 到 PA 的映射
